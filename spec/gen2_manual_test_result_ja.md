# GRSMD Gen2 手動テスト仕様書

**バージョン:** 1.0
**日付:** 2026-03-04
**対象:** `docs/index.html` (Vite ビルド出力)
**ブラウザ:** Chrome/Edge 86+ (FileSystemFileHandle テスト用)、Firefox (グレースフルデグレード確認用)

---

## 前提条件

| #    | 項目                       | 詳細                                                                   |
| ---- | -------------------------- | ---------------------------------------------------------------------- |
| P-01 | ビルドが最新であること     | `npm run build` を実行し、`docs/index.html` が生成されていることを確認 |
| P-02 | 自動テストが全パスすること | `npm test` を実行し、全107テストがパスすることを確認                   |
| P-03 | テスト用ファイルの準備     | 以下のファイルをローカルに用意する:                                    |
|      |                            | - `sample-data.md` (Mermaid + PlantUML + KaTeX を含む Markdown)        |
|      |                            | - `sample-data.txt` (プレーンテキスト)                                 |
|      |                            | - `sample-data-2.md` (別の Markdown ファイル)                          |
|      |                            | - 任意の `.py` ファイル (例: 短い Python スクリプト)                   |
|      |                            | - 任意の `.js` ファイル                                                |
|      |                            | - `Dockerfile` という名前のファイル (拡張子なし)                       |
|      |                            | - 未対応拡張子のファイル (例: `test.xyz`)                              |
|      |                            | - 5 MB 超のファイル (例: `big.bin`)                                    |
| P-04 | ブラウザ                   | Chrome または Edge (最新版)、OS のダーク/ライトモード切替が可能なこと  |
| P-05 | `docs/index.html` を開く   | ブラウザでファイルを直接開く (サーバー不要)                            |

### TC-02: Markdown ペースト (自動レンダリング)

|              | - 行番号が表示される                                            |
|              | - スティッキーヘッダーに `[CODE VIEW] filename.py               | タイムスタンプ | サイズ` が表示される |
|              | - 固定ステータスバーに `filename.py                             | N lines        | X.XX KB              | Loaded: HH:MM:SS` が表示される |
|              | - テーマがダークモードに強制される (body に `.dark` クラスあり) |
|              | - 画面下部にキーボードヒントオーバーレイが表示される            |
|              | - ヒントが約3秒後にフェードアウトする                           |

| 結果 | Pass / Fail / N/A            |
| ---- | ---------------------------- |
| 判定 | Pass                         |
| 備考 | 以下のUX改善の仕様変更が必要 |

1. [CODE VIEW] filename.py ... スティッキーヘッダーより エディター領域に出したい。
